## 4- Intro Kotlin

### Introdução ao Kotlin:

   - Iniciado em 2011 pela JetBrains.
   - Tornou-se a nova linguagem do Android em 2017.
   - Open Source - Apache 2.

### Vantagens e Características do Kotlin:

   - Tipagem dinâmica/inferência.
   - Type-safe e null-safe.
   - Compatível com Java.
   - Boa curva de aprendizado.
   - Desempenho equivalente ao Java.
   - Utilização de type casts seguros.
   - Capacidade de trabalhar com variáveis nulas.

### Por que desenvolver para Android com Kotlin?

   - Compatibilidade com JDK.
   - Desempenho através da geração de bytecodes.
   - Curva de aprendizado favorável.
   - Ambiente de desenvolvimento (IDE) necessário: IntelliJ IDEA Community Edition ou Kotlin Playground.

### REPL (READ-EVAL-PRINT LOOP):

   - Ambientes de desenvolvimento necessário.
   - Utilização de JDK do Java.
   - Estrutura padrão fornecida pelo Kotlin Playground.

### Arquivos Kotlin (.kt):

   - Extensão de arquivo padrão.
   - Comentários de uma única linha ou de múltiplas linhas.
   - Variáveis, constantes e tipos de dados.

### Variáveis e Constantes:

   - Declaração de variáveis com "var" e constantes com "val".
   - Tipagem dinâmica/inferência.
   - Declaração explícita do tipo.
   - Diferenças entre variáveis e constantes.

### Tipos de Dados Númericos em Kotlin:

   - Long, Int, Short, Byte, Double e Float.
   - Intervalos utilizáveis e tamanho em bits.
   - Uso de constantes para valores máximo e mínimo de tipos de dados.

### Imprimindo:

   - Dois tipos de impressão: println e print.
   - println: Pula automaticamente a linha após a impressão.
   - print: Não pula automaticamente a linha.

### String "texto":

   - Representa uma cadeia de caracteres.
   - Sempre delimitada por aspas.
   - Comando .length retorna o tamanho da string.
   - Concatenação com $Variavel ou com +.

### Tipos de texto:

   - Char: representa um único caractere, delimitado por aspas simples.
   - Pair: armazena dois valores diferentes em uma única variável.

### Booleanos:

   - Aceitam apenas dois estados: true e false.
   - Declarados com var ou val.

### Null Safety - Sufixo"?":

   - Evita NullPointerException.
   - Atribuição de valor inicial nulo.
   - Sufixo "?" indica possibilidade de valor nulo.

### Coleções Array e List:

   - Array: coleção imutável de tamanho fixo.
   - List: coleção mutável de tamanho variável.
   - Arrays podem ser inicializados de forma implícita ou explícita.
   - Funções úteis para Arrays e Lists: .isEmpty(), .size(), .count(), .add(), .contains(), .remove().

### Set:

   - Coleção que não permite itens repetidos.
   - Impressão em ordem alfabética.
   - Criando um HashSet.
   - Funções úteis: .size, .add, .contains, .remove.
   - Criando um SetOf().

### Map:

   - Lista chave-valor.
   - Criando um HashMap.
   - Funções úteis: .size, .put, .remove, .get.

### Operadores:

   - Atribuição: =.
   - Aritméticos: +, -, *, /, %.
   - Compostos: +=, -=, *=, /=, %=, ++, --.
   - Lógicos: &&, AND, ||, OR, !, NOT.
   - Comparação: >, <, >=, <=, ==, !=.

### Interpolação:

   - Juntar string e variável usando $.
   - Usar chaves para argumentos complexos.

### Estrutura de decisão:

   - if: condicional se.
   - else: condicional senão.
   - else if: condicional secundária.
   - if/else em uma linha.
   - when: similar ao switch.
   - when com múltiplas condições.
   - when com else.
   - when com múltiplos valores.

### Laços de repetição:

   - while: faça enquanto.
   - do while: faça enquanto (com execução inicial).
   - for: para (com incremento automático).
   - for para acessar itens de uma lista ou array.

### For com until:

   - Utiliza until para evitar ajustes no índice ao acessar o tamanho de uma lista.
   - Exemplo de uso com uma lista de números e verificação de números pares.

### Coalescência Nula (?:):

   - Operador ?: permite atribuir um valor padrão caso a variável seja nula.
   - Exemplo de uso com uma variável de idade.

### Close Range (..) e Half Closed Range (Until):

   - Criam intervalos de valores.
   - Closed Range (..) para intervalo aberto, Half Closed Range (until) para intervalo aberto no início e fechado no final.
   - Exemplo de uso com for para imprimir números de 1 a 10 e de 1 a 9.

### Enum:

   - Cria uma lista de valores fixos.
   - Os elementos devem ser escritos em letras maiúsculas.
   - Exemplo de declaração e uso de enum.

### Funções:

   - Usadas para modularizar o código e evitar repetições.
   - Devem ter responsabilidade única e não repetir código.
   - Exemplo de funções com e sem argumentos, retorno de valor e single expression.

### Filter (.filter):

   - Cria uma lista a partir de outra com filtros.
   - Itera sobre cada item da lista, adicionando à nova lista se a condição for atendida.
   - Exemplo de uso com uma lista de números e filtro de números pares.

### Map (.map):

   - Altera todos os itens de uma lista e os adiciona em uma nova lista.
   - Exemplo de uso para transformar uma lista de frutas em maiúsculas.

### Reduce (.reduce):

   - Reduz uma lista a um único item, aplicando uma operação de redução.
   - Exemplo de uso com uma lista de números para calcular a soma.

### Generics:

   - Utiliza it como um tipo genérico para representar qualquer tipo.

### Classes:

   - Definem propriedades, métodos e inicializadores.
   - Podem ser estendidas para expandir funcionalidades.
   - Trabalham com herança e type casting.

### Herança:

   - Uma classe herda propriedades de outra.
   - Filhos têm acesso aos métodos e atributos da mãe.
   - Exemplo de criação de classe mãe e filha com herança.

### Sobreescrita (Override):

   - Utilizada quando um filho herda um método da mãe, mas tem um comportamento diferente.
   - Necessário adicionar open na mãe e override no filho.

### Construtores de classes mãe e filha:

   - Constrói classes mãe e filha com construtores para inicializar atributos.
   - Exemplo de construção de classes com atributos e uso do construtor super

## 6- Intro ao Jetpack Compose

- Framework para UI Declarativa (toolkit) para desenvolvimento nativo de aplicativos Android.
- Apresentado no Google I/O 2019.
- Simplifica e acelera o desenvolvimento da Interface do Usuário (UI) no Android.
- Anteriormente, a criação de interfaces era feita em XML, mas o Jetpack Compose simplifica isso ao permitir a definição da UI em Kotlin.

### Composable
- Componentes que compõem a tela.
- Diminui erros de atualização.
- Permite fazer mais com menos código.
- Função em Kotlin que define a UI de um componente específico da tela.
- Possui uma variedade de composables pré-construídos para criar a interface da aplicação.

### State
- Controla o estado da UI.
- Declarativo e reativo.
- Representa os dados que afetam a aparência ou o comportamento dos componentes da interface.

### O Novo e o Antigo
- No desenvolvimento tradicional, os componentes eram construídos com XML, o que aumentava a complexidade.
- Com Jetpack Compose, tudo fica em um único arquivo Kotlin, simplificando o desenvolvimento.

### Criação de um Projeto
- Funções @Composable usam a primeira letra maiúscula.
- Limpar o conteúdo padrão da função principal.
- Criar componentes com @Composable fora da função principal.
- Adicionar o @Preview do componente.
- Chamar o componente na função principal.

### Estilizando Textos
- Parâmetros como fontSize, color e fontWeight podem ser utilizados para estilizar textos.
- Utilização de fontes escaláveis (sp).

### Modificando Botões
- Parâmetro modifier para aplicar modificações aos composables.
- Parâmetro shape para modificar a forma do Button.
- Parâmetro colors para modificar a cor do botão.

### Alinhando o Conteúdo
- Parâmetros horizontalAlignment e verticalArrangement para alinhar o conteúdo da Column.

### Espaçamento
- Utilização do Spacer para adicionar espaçamento entre os elementos.
- Parâmetros height e width para espaçamento vertical e horizontal, respectivamente.

### Demais Parâmetros de Text
- Existem outros parâmetros para personalizar o texto.

### Definindo Comportamento da Aplicação - State
- Utilização de variáveis de estado para controlar o comportamento da aplicação.
- Implementação do método onClick para incrementar ou decrementar valores.

### Desafio
1. Adicionar um texto abaixo dos botões que exiba a mensagem informando se o usuário é maior ou menor de idade.
2. Impedir que o valor de idade seja menor do que zero ou maior do que 130.

## 7- Layout Componentes basics

- O Jetpack Compose facilita a criação de interfaces de usuário (UI) de forma produtiva.
- O sistema de Layouts do Jetpack Compose fornece vários elementos para criar UIs intuitivas.

### Box e Surface

- `Box` é usado para agrupar composables dentro de uma área retangular.
- Comparado a uma DIV no HTML, o `Box` é um container de composables que pode posicionar os elementos de acordo com a orientação dada.
- O parâmetro `contentAlignment` da `Box` permite alinhar o conteúdo em qualquer lado da Box.

### Alinhando componentes na Box

- O uso de `contentAlignment` alinha todos os composables no mesmo lugar, mas podemos utilizar o parâmetro `modifier` nos composables filhos para ajustar suas posições dentro da `Box`.

### Controle de posicionamento com "OFFSET"

- É possível mover os componentes dentro da `Box` utilizando coordenadas x e y.
- Coordenadas negativas também são suportadas, mas recomenda-se envolvê-las em parênteses.

### Column e Row

- São os composables de layout mais utilizados no Jetpack Compose.
- Permitem posicionar componentes lado a lado, na horizontal ou vertical, conforme inseridos no código.

### Modifier

- O `Modifier` é uma classe que permite modificar os estilos de um composable, como tamanho, cor, posicionamento, etc.
- As modificações são aplicadas na ordem em que são declaradas.

### Alinhamento de Column e Row

- Parâmetros como `horizontalAlignment`, `verticalAlignment`, `horizontalArrangement` e `verticalArrangement` permitem ajustar o posicionamento e o arranjo dos composables dentro de Columns e Rows.

## 8- Cards e imagens

- As aplicações frequentemente utilizam imagens para apresentar conteúdos ao usuário.
- Os Cards oferecem uma maneira fácil de agrupar e representar informações relacionadas em blocos.

### Apresentação do projeto

- O projeto consiste em criar um aplicativo para cálculo do Índice de Massa Corporal (IMC).
- Utiliza componentes como "Image" e "Card" para a interface do usuário.

### Estrutura do layout

- O layout é estruturado com os seguintes componentes:
  - Surface: envolve toda a atividade.
  - Box: ocupa todo o espaço da atividade, facilitando o posicionamento do "Card" com o resultado na parte inferior.
  - Column1: organiza verticalmente a Column2 (cabeçalho) e a Column3 (formulário).
  - Column2: cabeçalho da atividade.
  - Column3: formulário para entrada de dados.
  - Card Formulário: estiliza o formulário.
  - Card Resultado: exibe o resultado do cálculo do IMC.

### Componente Image

- Permite exibir imagens com diferentes parâmetros, como:
  - Painter: aponta para a imagem desejada.
  - Modifier: permite modificar aspectos como tamanho e escala da imagem.
  - ContentScale: ajusta a escala da imagem para preencher o espaço disponível.

### Cores definidas em colors.xml

- As cores são definidas no arquivo colors.xml para serem utilizadas no projeto.
- São acessadas usando `colorResource(id = R.color.nome_da_cor)`.

### Cards

- Usados para organizar o conteúdo da aplicação em grupos.
- Possuem propriedades como modifier, shape, cores e elevação.

### Funções

- São criadas funções para atribuir valores aos campos do formulário e para calcular o IMC.
- Uma função adicional é criada para determinar o status do IMC (abaixo do peso, peso ideal, etc.).

### Implementação do projeto

- O código é estruturado utilizando composables do Jetpack Compose.
- São utilizados modifiers para definir o layout e o estilo dos componentes.
- O resultado do cálculo do IMC e seu status são exibidos na interface do usuário.

## 11 Internacionalização de Aplicativos

- Escopo Global: Ao desenvolver um aplicativo, é crucial considerar seu alcance potencial, seja regional, nacional ou internacional.
- Idioma do Dispositivo: O aplicativo deve detectar automaticamente o idioma principal do dispositivo e adaptar-se a ele.
- Definição de Idioma: O idioma do dispositivo é configurado em "Configurações" > "Sistema" > "Idiomas".
- String.xml: As frases do aplicativo são armazenadas no arquivo "string.xml" dentro da pasta do projeto para facilitar a tradução.
- Tradução Simplificada: Ao traduzir o aplicativo, apenas os campos do arquivo "string.xml" precisam ser traduzidos, em vez de editar diretamente todos os campos.
- Arquivos Separados por Idioma: Cada idioma tem seu próprio arquivo "string.xml" para manter as traduções organizadas.

### Validação de Entrada de Dados:

- Prevenção de Entrada Incorreta: A validação de entrada de dados é importante para garantir que os usuários insiram informações corretas.
- Formas de Validação: Existem várias técnicas de validação, como entregar o teclado apropriado para o tipo de entrada ou validar ao clicar no campo.
- Exemplo de Validação Simples: Um exemplo comum é limitar o número máximo de caracteres em um campo de senha.
- Indicação de Erro ao Usuário: O componente `OutlinedTextField` possui um atributo `isError` que indica visualmente quando há um erro na entrada do usuário.
- Feedback ao Usuário: Mensagens de erro informativas podem ser exibidas para orientar o usuário sobre o que foi inserido de forma incorreta.
- Tratamento de Campo Vazio: Testes podem ser feitos para verificar se um campo está vazio e informar o usuário sobre a obrigatoriedade de preenchê-lo.
